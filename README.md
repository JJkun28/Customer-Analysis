# 🛒 电商新客获客质量与 ROI 分析看板

#### 📊 项目简介
本项目是一个端到端的**全链路数据分析实战**。
项目基于**2025年1月新注册用户群组**的全生命周期数据，模拟了从数据生成、清洗建模到商业可视化的完整流程。
观测窗口覆盖 2025年1月至4月，再通过对**AARRR漏斗模型**的量化分析，诊断各渠道流量质量，并制定预算优化策略。

**核心成果：**
通过对 5 个核心渠道的深度诊断，识别出**高价值** 与**低效**渠道，提出了**缩减 30% 低效预算**并**启动裂变激励**的战略建议，预计可显著提升整体 LTV 与投放效能。

#### 💡 核心商业洞察

基于2025年1月新注册用户群组数据的多维诊断，得出以下战略建议：

**1. 收入端(Revenue)：利用激励机制突破 Referral 传播瓶颈**
* **数据表现**：`Friend_Referral` (好友推荐) 渠道展现出极高的用户质量，其 **激活率 (79.3%)**、**付费转化率 (24.7%)** 及 **LTV (¥18.6)** 均断层领先于其他渠道。然而，其 **病毒系数(K-Factor)** 仅为 **0.1** < 1，表明自然裂变效应微弱。
* **战略判断**：该渠道是目前 **单用户价值** 最高的增长点，受限于传播动力不足。
* **行动建议**：建议立即启动 **双向现金奖励** 机制。鉴于其极高的 LTV 表现，我们在拥有充足的CPA (获客成本)空间时，应通过强激励手段打破用户的社交分享门槛，激活存量用户带动增量。

**2. 获客端(Acquisition)：针对 TikTok 实施流量分层与促活转化**
* **数据表现**：`TikTok_Ads`贡献了 30% 的最大流量规模。值得注意的是，其**平均次日留存率 (45.5%)** 与 **AppStore(自然流量)** 保持在同一水平，这客观证明了该渠道引入的用户并非劣质流量，而是对产品具有较好的基础粘性。
* **战略判断**：然而，其 **付费转化率(2.6%)** 与 **LTV(¥1.8)** 均为全渠道最低。这表明 TikTok 用户呈现典型的 **“高活跃、低付费”** 特征，用户价值未被充分挖掘。
* **行动建议**：
    * **投放侧**：缩减纯展示类预算，优化人群包，剔除低意愿用户。
    * **运营侧**：针对该渠道来源用户，设计低门槛的 **首单优惠** 活动，优先解决付费转化率过低的问题。

**3. 激活端(Activation)：对Baidu SEM进行精细化运营**
* **数据表现**：`Baidu_SEM` 的指标呈现显著的两极分化——**激活率 (40.5%)** 与 **平均次日留存率 (38.1%)** 全场最低，反映出流量精准度不足；但其 **ARPPU (每付费用户平均收益)** 高达 **¥93.0**，位居全渠道之首。
* **战略判断**：SEM 渠道虽然引入了大量非目标用户（导致留存低），但筛选出的核心用户具备极强的消费能力。
* **行动建议**：不建议直接关停。应重点排查并剔除高消耗、低转化的“宽泛匹配”关键词，将预算集中于高转化的“精准长尾词”，以提升整体 **投放效能**。

---

#### 🔍 深度归因：同留存下的价值差异分析

* **现象观测**：
    对比分析发现，`TikTok` 与 `AppStore` 拥有惊人相似的 **次日留存率 (均在 45% 左右)**，表明两个渠道的用户在“留存意愿”上并无显著差异。然而，`AppStore` 的 LTV (¥4.84) 却是 `TikTok` (¥1.80) 的 **2.7倍**。

* **逻辑归因**：
    通过拆解 LTV 公式 LTV = 付费转化率 × ARPPU，差异的核心驱动力在于：
    1.  **转化能力差异**：AppStore 的付费转化率 (5.5%) 是 TikTok (2.6%) 的两倍以上。
    2.  **消费力差异**：AppStore 的 ARPPU (¥88.8) 显著高于 TikTok (¥70.2)。

* **结论**：
    这一数据揭示了用户画像的本质区别：`TikTok` 用户主要为**价格敏感型群体**，而 `AppStore` 用户则具备**更强的付费意愿与消费能力**。战略上，应继续以 AppStore 作为核心利润基准，同时对 TikTok 进行针对性的变现策略调整。

#### 🛠️ 技术架构与工作流

本项目分为三个阶段，模拟了真实企业环境下的数据流转过程：

**阶段1: 数据模拟**
使用 **Python** 模拟生成了符合业务逻辑的原始业务表(raw data)。
* `user_data.csv`: 用户基础信息（UserID、注册时间、渠道 等）。
* `order_data.csv`: 交易流水记录（订单时间、金额、关联用户）。
* `login_data.csv`: 用户每日登录日志（用于计算高频留存指标）。
*(注：数据逻辑包含了不同渠道的差异化留存与付费参数，以模拟真实商业环境。)*

**阶段2: 数据建模**
利用 SQL 对原始数据进行清洗、聚合与指标计算，构建了基于 **AARRR 模型** 的数据中间层。
